# Subjunctive Mood & Formal Demands

In technical documentation, pull request reviews, and emails to clients or managers, you often need to express requirements, recommendations, or urgent requests. To sound professional and grammatically correct, you must use the **Subjunctive Mood**.

---

## 1. What is the Subjunctive Mood?

The subjunctive mood is used when expressing a wish, demand, suggestion, or requirement. 
- **The Golden Rule:** The verb in the `that` clause is **always in its base form** (bare infinitive). It does not change based on the subject (no `-s`, `-es`, or past tense). Even the verb `be` remains `be` (not *am/is/are/was/were*).

- ❌ *Wrong:* "It is important that he **updates** the database config."
- 🟢 *Correct:* **"It is important that he update the database config."**
- ❌ *Wrong:* "We recommend that the team **deploys** the hotfix now."
- 🟢 *Correct:* **"We recommend that the team deploy the hotfix now."**
- ❌ *Wrong:* "The client requested that the logo **is** changed."
- 🟢 *Correct:* **"The client requested that the logo be changed."**

---

## 2. Common Adjectives Triggering the Subjunctive

Use this formula: `It is + Adjective + that + Subject + Base Verb`

### 💡 10 Practical Examples:

1.  **"It is critical that the server remain online during the migration."** (Not: *remains*)
2.  **"It is necessary that the user reset their password after the first login."** (Not: *resets*)
3.  **"It is mandatory that every developer write unit tests for their code."** (Not: *writes*)
4.  **"It is essential that the build configuration be checked before release."** (Not: *is*)
5.  **"It is vital that all user data be encrypted at rest."** (Not: *is encrypted*)
6.  **"It is important that he configure the environment variables correctly."** (Not: *configures*)
7.  **"It is required that the applicant possess basic knowledge of Docker."** (Not: *possesses*)
8.  **"It is imperative that the cache be cleared after deployment."** (Not: *is cleared*)
9.  **"It is urgent that we patch the security vulnerability in the auth route."** (Not: *we patch* - here *patch* is the base form, which happens to look like the normal form).
10. **"It is recommended that the script run in a sandbox environment."** (Not: *runs*)

---

## 3. Common Verbs Triggering the Subjunctive

Use this formula: `Subject + Verb + that + Subject + Base Verb`

### 💡 10 Practical Examples:

1.  **"I recommend that Huy review the security policy."** (Not: *reviews*)
2.  **"The architect suggested that we shard the database next year."** (Not: *we shard* - here *shard* is in base form).
3.  **"The client insists that the app support offline mode."** (Not: *supports*)
4.  **"The system requires that the admin approve the transaction manually."** (Not: *approves*)
5.  **"DevOps requested that the development team reduce the container size."** (Not: *reduces*)
6.  **"We ask that each engineer submit their weekly report by Friday."** (Not: *submits*)
7.  **"The security audit demanded that we rotate all API secrets."** (Not: *we rotated*)
8.  **"I suggest that she test the endpoint using Postman first."** (Not: *tests*)
9.  **"The project manager proposed that we extend the sprint by three days."** (Not: *we extend* - base form).
10. **"The license agreement demands that we credit the original author."** (Not: *we credit* - base form).

---

## 4. Negative Subjunctive (Not + Base Verb)

To make a subjunctive sentence negative, simply put **not** directly before the base verb (do **NOT** use *does not*, *do not*, or *did not*).

- ❌ *Wrong:* "It is vital that we don't expose the API key."
- 🟢 *Correct:* **"It is vital that we not expose the API key."**
- ❌ *Wrong:* "I recommend that the script doesn't run automatically."
- 🟢 *Correct:* **"I recommend that the script not run automatically."**

### 💡 10 Practical Examples:

1.  **"It is critical that the team not push raw secrets to GitHub."**
2.  **"I suggest that he not merge the PR until tests are green."**
3.  **"The client asked that the payment service not save credit card details."**
4.  **"It is necessary that the daemon service not block the main process."**
5.  **"I recommend that we not delete the backup logs yet."**
6.  **"It is crucial that the server not restart during active backups."**
7.  **"We require that external contributors not access the production repo."**
8.  **"It is essential that this flag not be set to true in production."**
9.  **"I suggest that the app not send email notifications to offline users."**
10. **"The guidelines demand that developers not hardcode configurations."**

## 5. Summary Cheat Sheet

| Trigger Phrase | Example Sentence | Meaning |
| :--- | :--- | :--- |
| **It is crucial that...** | "It is crucial that the lead developer **sign** off on this." | Trưởng nhóm cần ký duyệt cái này. |
| **I recommend that...** | "I recommend that he **not pull** the changes yet." | Tôi khuyên anh ấy chưa nên pull code mới về. |
| **The client asks that...**| "The client asks that the design **be** simplified." | Khách hàng yêu cầu thiết kế được đơn giản hóa. |
| **It is essential that...** | "It is essential that this field **not be** empty." | Cần thiết là trường này không được để trống. |
| **They insist that...** | "They insist that he **explain** the delay." | Họ khăng khăng yêu cầu cậu ấy giải thích việc trễ hạn. |
