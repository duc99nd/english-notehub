# Common Phrasal Verbs in IT

Phrasal verbs are two- or three-word combinations (verb + preposition/adverb) that have a specific meaning different from the individual words. Native English speakers use them constantly in meetings, Slack messages, and code reviews. Mastering IT phrasal verbs will make your English sound natural and fluent.

---

## 1. Development & Coding

### 💡 10 Essential Phrasal Verbs:

1.  **set up** — To configure or prepare something for use.
    - *"Can you **set up** the local development environment?"*
2.  **spin up** — To start or launch (a server, container, instance).
    - *"Let me **spin up** a new Docker container for testing."*
3.  **tear down** — To dismantle or remove infrastructure.
    - *"We need to **tear down** the staging environment after the demo."*
4.  **hook up** — To connect two systems or services.
    - *"I **hooked up** the frontend to the new REST API."*
5.  **break down** — To divide into smaller parts; also means to stop working.
    - *"Let's **break down** this user story into subtasks."*
    - *"The server **broke down** during peak traffic."*
6.  **plug in** — To add a component or integration.
    - *"We just need to **plug in** the payment gateway module."*
7.  **opt in / opt out** — To choose to participate / not participate.
    - *"Users can **opt out** of email notifications in their settings."*
8.  **figure out** — To solve or understand something.
    - *"I need to **figure out** why the API returns a 403 error."*
9.  **comment out** — To disable code by wrapping it in comments.
    - *"I **commented out** the debug logs before pushing to production."*
10. **refactor out** — To remove unnecessary code through refactoring.
    - *"We should **refactor out** the duplicated validation logic."*

## 2. Deployment & Operations

### 💡 10 Essential Phrasal Verbs:

1.  **roll back** — To revert to a previous version.
    - *"The hotfix failed. We need to **roll back** to version 1.4.2."*
2.  **roll out** — To deploy or release a new version.
    - *"We will **roll out** the new feature to 10% of users first."*
3.  **scale up / scale out** — To increase resources (vertically/horizontally).
    - *"We need to **scale up** the database server to handle more queries."*
4.  **shut down** — To stop a service or server.
    - *"Please **shut down** the staging server before the maintenance window."*
5.  **back up** — To create a copy of data for safety.
    - *"Make sure you **back up** the database before running the migration."*
6.  **switch over** — To change from one system to another.
    - *"We will **switch over** to the new CDN provider at midnight."*
7.  **phase out** — To gradually stop using something.
    - *"We are **phasing out** support for Node 14 by the end of Q3."*
8.  **boot up** — To start a machine or system.
    - *"The server takes 45 seconds to **boot up** after a restart."*
9.  **log in / log out** — To authenticate / end a session.
    - *"Users are unable to **log in** after the latest deployment."*
10. **patch up** — To fix temporarily.
    - *"Let's **patch up** the security hole tonight and do a proper fix next sprint."*

## 3. Communication & Meetings

### 💡 10 Essential Phrasal Verbs:

1.  **follow up** — To check on progress after a discussion.
    - *"I will **follow up** with the client about the API credentials."*
2.  **bring up** — To introduce a topic in a meeting.
    - *"I want to **bring up** the database bottleneck in today's standup."*
3.  **point out** — To highlight or indicate.
    - *"Huy **pointed out** that the SSL certificate expires next week."*
4.  **go over** — To review or examine.
    - *"Let's **go over** the deployment checklist before we push to production."*
5.  **wrap up** — To finish or conclude.
    - *"Let's **wrap up** this meeting. We've covered all the action items."*
6.  **circle back** — To return to a topic later.
    - *"Let's **circle back** on the caching strategy after lunch."*
7.  **sign off** — To approve formally.
    - *"The client needs to **sign off** on the design before we start coding."*
8.  **hold off** — To delay or postpone.
    - *"Let's **hold off** on the migration until the performance tests pass."*
9.  **run by** — To share an idea for feedback.
    - *"Can I **run** this architecture proposal **by** you before the meeting?"*
10. **catch up** — To get updated on progress.
    - *"I was on leave. Can we **catch up** on what happened during the sprint?"*
