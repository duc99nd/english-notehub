# Thức Giả Định trong Yêu Cầu & Đề Xuất (Daily English)

Trong tài liệu kỹ thuật, bình luận review PR (pull request) và email gửi cho khách hàng hoặc sếp, bạn thường xuyên phải đưa ra các đề xuất, yêu cầu bắt buộc hoặc các giải pháp khẩn cấp. Để câu nói nghe trang trọng và đúng ngữ pháp, bạn cần sử dụng **Thức giả định (Subjunctive Mood)**.

---

## 1. Thức Giả Định là gì?

Thức giả định được dùng khi muốn thể hiện một mong muốn, đề xuất, yêu cầu hoặc một điều bắt buộc.
- **Quy tắc vàng:** Động từ đứng sau mệnh đề chứa `that` **luôn ở dạng nguyên mẫu không chia (base form / bare infinitive)**. Nó hoàn toàn không thay đổi theo chủ ngữ (không thêm `-s`, `-es` và không chia ở thì quá khứ).

- ❌ *Sai:* "It is important that he **updates** the database config."
- 🟢 *Đúng:* **"It is important that he update the database config."** *(Quan trọng là anh ấy phải cập nhật file cấu hình database).*
- ❌ *Sai:* "We recommend that the team **deploys** the hotfix now."
- 🟢 *Đúng:* **"We recommend that the team deploy the hotfix now."** *(Chúng tôi khuyến nghị team nên deploy bản hotfix ngay bây giờ).*

---

## 2. Các Tính Từ Thường Dùng Với Thức Giả Định

Công thức: `It is + Tính từ + that + Chủ ngữ + Động từ nguyên mẫu`

- **crucial / critical / essential (cực kỳ quan trọng):**
  - **"It is critical that the server remain online during the migration."** *(Điều cốt lõi là server phải duy trì online trong quá trình migrate - Dùng remain, không dùng remains).*
- **important / necessary (quan trọng / cần thiết):**
  - **"It is necessary that the user reset their password after the first login."** *(Cần thiết là người dùng phải reset lại mật khẩu sau lần đăng nhập đầu tiên).*
- **mandatory / required (bắt buộc):**
  - **"It is mandatory that every developer write unit tests for their code."** *(Bắt buộc mỗi dev phải viết unit test cho code của mình).*

---

## 3. Các Động Từ Thường Dùng Với Thức Giả Định

Công thức: `Chủ ngữ 1 + Động từ + that + Chủ ngữ 2 + Động từ nguyên mẫu`

- **recommend / suggest (đề xuất/khuyên dùng):**
  - **"I recommend that Huy review the security policy."** *(Tôi đề xuất Huy nên rà soát lại chính sách bảo mật - Dùng review, không dùng reviews).*
  - **"The architect suggested that we shard the database next year."**
- **demand / insist (đòi hỏi/quả quyết):**
  - **"The client insists that the app support offline mode."** *(Khách hàng khăng khăng yêu cầu app phải hỗ trợ chế độ offline).*
- **require / ask (yêu cầu):**
  - **"The system requires that the admin approve the transaction."**

---

## 4. Dạng Phủ Định của Thức Giả Định (Not + Động từ nguyên mẫu)

Để viết câu phủ định, bạn chỉ cần đặt chữ **not** trực tiếp trước động từ nguyên mẫu (tuyệt đối **KHÔNG** dùng *does not*, *do not* hoặc *did not*).

- ❌ *Sai:* "It is vital that we don't expose the API key."
- 🟢 *Đúng:* **"It is vital that we not expose the API key."** *(Điều tối quan trọng là chúng ta không được làm lộ API key).*
- 🟢 **"I recommend that the script not run automatically."** *(Tôi khuyên không nên cho script này chạy tự động).*

---

## 5. Bảng Tra Cứu Nhanh

| Cụm từ kích hoạt | Câu ví dụ | Nghĩa tiếng Việt |
| :--- | :--- | :--- |
| **It is crucial that...** | "It is crucial that the lead developer **sign** off on this." | Cực kỳ quan trọng là lead developer phải ký duyệt cái này. |
| **I recommend that...** | "I recommend that he **not pull** the changes yet." | Tôi khuyên anh ấy không nên kéo code mới về lúc này. |
| **The client asks that...** | "The client asks that the design **be** simplified." | Khách hàng yêu cầu giao diện phải được tối giản hóa. |
