# Giới Từ Trong Ngữ Cảnh IT (Daily English)

Giới từ (in, on, at, to, for, with, by, from) là những từ nhỏ nhưng gây ra các lỗi lớn. Lập trình viên Việt Nam thường xuyên chọn sai giới từ vì logic giới từ tiếng Việt không dịch trực tiếp sang tiếng Anh được. Hướng dẫn này bao gồm các cụm giới từ phổ biến nhất trong IT.

---

## 1. Giới Từ Với Các Hành Động Kỹ Thuật

### 💡 10 Cụm từ "Deploy / Push / Commit":

1.  **deploy TO** — "We **deploy to** production every Friday." *(Chúng tôi deploy lên production mỗi thứ Sáu).*
2.  **push TO** — "Please **push to** the main branch." *(Vui lòng push code lên nhánh main).*
3.  **commit TO** — "I **committed** the fix **to** the feature branch." *(Tôi đã commit bản sửa lỗi vào nhánh feature).*
4.  **merge INTO** — "We **merged** the hotfix **into** main." *(Chúng tôi đã merge hotfix vào nhánh main).*
5.  **pull FROM** — "**Pull** the latest changes **from** the remote repository." *(Pull các thay đổi mới nhất từ remote repo).*
6.  **connect TO** — "The app cannot **connect to** the database." *(App không thể kết nối tới database).*
7.  **migrate TO** — "We are **migrating to** PostgreSQL." *(Chúng tôi đang migrate sang PostgreSQL).*
8.  **switch TO** — "Let's **switch to** the staging environment." *(Hãy chuyển sang môi trường staging).*
9.  **upgrade TO** — "We **upgraded to** Node 20." *(Chúng tôi đã nâng cấp lên Node 20).*
10. **revert TO** — "We need to **revert to** the previous release." *(Chúng ta cần revert về bản release trước đó).*

### 💡 10 Cụm từ "Work / Depend / Focus":

1.  **work ON** — "I am **working on** the authentication module." *(Tôi đang làm việc trên module xác thực — Không nói: work in).*
2.  **depend ON** — "This feature **depends on** the user service API." *(Tính năng này phụ thuộc vào API user service).*
3.  **focus ON** — "Let's **focus on** fixing the critical bugs." *(Hãy tập trung vào việc sửa các bug nghiêm trọng).*
4.  **rely ON** — "We **rely on** Redis for session management." *(Chúng tôi dựa vào Redis để quản lý session).*
5.  **run ON** — "This application **runs on** Docker containers." *(Ứng dụng này chạy trên các container Docker).*
6.  **based ON** — "The decision was **based on** performance benchmarks." *(Quyết định dựa trên các kết quả benchmark hiệu năng).*
7.  **impact ON** — "This change has a major **impact on** loading speed." *(Thay đổi này có ảnh hưởng lớn đến tốc độ tải).*
8.  **work WITH** — "We **work with** the design team on UI." *(Chúng tôi làm việc với team thiết kế về UI).*
9.  **integrate WITH** — "Our system **integrates with** Stripe for payments." *(Hệ thống của chúng tôi tích hợp với Stripe để thanh toán).*
10. **compatible WITH** — "This plugin is **compatible with** version 3.x." *(Plugin này tương thích với phiên bản 3.x).*

## 2. Giới Từ Với Thời Gian & Lịch Họp

### 💡 10 Ví dụ thực tế:

1.  **AT** + thời điểm cụ thể — "The standup is **at** 9:30 AM." *(Standup lúc 9:30 sáng).*
2.  **ON** + ngày cụ thể — "We deploy **on** Fridays." *(Chúng tôi deploy vào các ngày thứ Sáu).*
3.  **IN** + khoảng thời gian — "The fix will be ready **in** two hours." *(Bản sửa lỗi sẽ xong trong 2 giờ nữa).*
4.  **BY** + thời hạn chót — "Please finish this **by** end of day." *(Vui lòng hoàn thành trước cuối ngày).*
5.  **DURING** + trong suốt — "The server crashed **during** peak hours." *(Server bị sập trong giờ cao điểm).*
6.  **BEFORE** + trước — "Run the tests **before** deployment." *(Chạy test trước khi deploy).*
7.  **AFTER** + sau — "We'll discuss this **after** the retro." *(Chúng ta sẽ bàn việc này sau buổi retro).*
8.  **WITHIN** + trong vòng — "Reply **within** 24 hours." *(Trả lời trong vòng 24 giờ).*
9.  **SINCE** + kể từ — "The bug has existed **since** last Tuesday." *(Bug này đã tồn tại kể từ thứ Ba tuần trước).*
10. **UNTIL** + cho tới — "The feature freeze lasts **until** Friday." *(Giai đoạn đóng băng tính năng kéo dài đến thứ Sáu).*

## 3. Giới Từ Với Hệ Thống & Vị Trí

### 💡 10 Ví dụ thực tế:

1.  **IN** + file — "There is a typo **in** the config file." *(Có lỗi chính tả trong file cấu hình).*
2.  **IN** + môi trường — "This works **in** staging but not **in** production." *(Cái này chạy trên staging nhưng không chạy trên production).*
3.  **ON** + server — "The app is running **on** our AWS server." *(App đang chạy trên server AWS của chúng ta).*
4.  **ON** + nhánh — "I am **on** the feature branch." *(Tôi đang ở trên nhánh feature).*
5.  **ON** + trang/màn hình — "The button **on** the login page is broken." *(Nút trên trang đăng nhập bị hỏng).*
6.  **AT** + URL — "The documentation is available **at** this URL." *(Tài liệu có sẵn tại URL này).*
7.  **FROM** + nguồn — "Download the package **from** npm." *(Tải package từ npm).*
8.  **INTO** + đích — "Import the data **into** the database." *(Import dữ liệu vào database).*
9.  **THROUGH** + qua — "The request passes **through** the middleware." *(Request đi qua middleware).*
10. **ACROSS** + xuyên suốt — "We need data consistency **across** all microservices." *(Chúng ta cần tính nhất quán dữ liệu xuyên suốt các microservice).*

## 4. Các Lỗi Giới Từ Phổ Biến

### ❌ 10 Lỗi Mà Dev Việt Nam Hay Mắc:

1.  ❌ "I will deploy **in** production." → 🟢 "I will deploy **to** production." *(Deploy lên, không phải deploy trong)*
2.  ❌ "I am working **in** this task." → 🟢 "I am working **on** this task." *(Làm việc trên task, không phải trong task)*
3.  ❌ "The meeting is **in** Monday." → 🟢 "The meeting is **on** Monday." *(Dùng ON với ngày)*
4.  ❌ "I depend **with** this library." → 🟢 "I depend **on** this library." *(Phụ thuộc vào, không phải với)*
5.  ❌ "Please push **in** the main branch." → 🟢 "Please push **to** the main branch."
6.  ❌ "We discussed **about** the bug." → 🟢 "We discussed the bug." *(Discuss không cần giới từ)*
7.  ❌ "The error happened **in** 3 PM." → 🟢 "The error happened **at** 3 PM." *(Dùng AT với giờ cụ thể)*
8.  ❌ "I need to connect **with** the database." → 🟢 "I need to connect **to** the database."
9.  ❌ "This is compatible **to** version 2." → 🟢 "This is compatible **with** version 2."
10. ❌ "I will finish this task **on** two hours." → 🟢 "I will finish this task **in** two hours." *(Dùng IN với khoảng thời gian)*
