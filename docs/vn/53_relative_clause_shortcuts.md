# Mẹo Rút Gọn Mệnh Đề Quan Hệ (Daily English)

Mệnh đề quan hệ được dùng để bổ sung chi tiết cho một danh từ (ví dụ: một con bug, một thư viện, hay một lập trình viên) mà không cần phải ngắt câu để viết câu mới. Việc sử dụng **rút gọn** mệnh đề quan hệ (reductions) là mẹo dễ nhất để bạn nói tiếng Anh ngắn gọn, tự nhiên như người bản xứ.

---

## 1. Các Đại từ Quan hệ Cốt lõi trong IT

Trước khi học mẹo rút gọn, hãy nhớ lại 5 đại từ quan hệ cơ bản sau:
- **that / which** (dùng cho công cụ, thư viện, tính năng): *"We fixed the bug **that** crashed the system."* *(Chúng tôi đã sửa con bug làm sập hệ thống).*
- **who** (dùng cho đồng nghiệp, user, khách hàng): *"The engineer **who** designed this is on leave."* *(Kỹ sư thiết kế cái này đang nghỉ phép).*
- **where** (dùng cho trang màn hình, vị trí, database): *"This is the screen **where** users change passwords."* *(Đây là màn hình nơi user đổi mật khẩu).*
- **when** (dùng cho sự kiện, thời điểm, đợt release): *"That was the time **when** our traffic spiked."* *(Đó là thời điểm lượng traffic của chúng ta tăng vọt).*

---

## 2. Mẹo 1: Rút gọn Chủ động (Dùng đuôi V-ing)

Nếu động từ trong mệnh đề quan hệ ở thể **chủ động** (danh từ đứng trước tự thực hiện hành động đó), bạn có thể bỏ đại từ quan hệ (*who/which/that*) và chuyển động từ chính về dạng thêm đuôi **-ing**.

- **Công thức:** `Danh từ + who/which/that + Động từ` ➔ `Danh từ + V-ing`

### 💡 10 Ví dụ thực tế:

1.  *Đầy đủ:* "We found a bug **that blocks** the checkout page."
    ➔ **Rút gọn: "We found a bug blocking the checkout page."** *(Chúng tôi tìm thấy một con bug chặn trang checkout).*
2.  *Đầy đủ:* "We need a tool **that generates** clean PDF reports."
    ➔ **Rút gọn: "We need a tool generating clean PDF reports."** *(Chúng tôi cần một công cụ tạo ra báo cáo PDF sạch đẹp).*
3.  *Đầy đủ:* "The developer **who works** on this feature is out today."
    ➔ **Rút gọn: "The developer working on this feature is out today."** *(Developer làm tính năng này hôm nay nghỉ).*
4.  *Đầy đủ:* "The script **that updates** user accounts is failing."
    ➔ **Rút gọn: "The script updating user accounts is failing."** *(Script cập nhật tài khoản người dùng đang bị lỗi).*
5.  *Đầy đủ:* "We noticed a background service **that consumes** 90% CPU."
    ➔ **Rút gọn: "We noticed a background service consuming 90% CPU."** *(Chúng tôi phát hiện một dịch vụ ngầm ngốn tới 90% CPU).*
6.  *Đầy đủ:* "Any query **that bypasses** the cache will slow down the page."
    ➔ **Rút gọn: "Any query bypassing the cache will slow down the page."** *(Bất kỳ câu query nào bỏ qua cache sẽ làm chậm trang).*
7.  *Đầy đủ:* "We have multiple users **who complain** about the new payment layout."
    ➔ **Rút gọn: "We have multiple users complaining about the new payment layout."** *(Chúng tôi có nhiều user phàn nàn về layout thanh toán mới).*
8.  *Đầy đủ:* "Is there an index **that speeds** up this join query?"
    ➔ **Rút gọn: "Is there an index speeding up this join query?"** *(Có index nào giúp tăng tốc câu query join này không?)*
9.  *Đầy đủ:* "I am looking for a package **that supports** WebSocket connections."
    ➔ **Rút gọn: "I am looking for a package supporting WebSocket connections."** *(Tôi đang tìm một package hỗ trợ kết nối WebSocket).*
10. *Đầy đủ:* "The server **that hosts** the staging database needs a reboot."
    ➔ **Rút gọn: "The server hosting the staging database needs a reboot."** *(Server host database staging cần được restart).*

## 3. Mẹo 2: Rút gọn Bị động (Dùng V3/ed)

Nếu động từ trong mệnh đề quan hệ ở thể **bị động** (danh từ đứng trước bị/được tác động, câu thường có cấu trúc `be + V3/ed`), bạn có thể bỏ cả đại từ quan hệ (*who/which/that*) lẫn động từ `be`, chỉ giữ lại động từ ở cột 3 hoặc đuôi **-ed**.

- **Công thức:** `Danh từ + who/which/that + is/are/was/were + V3/ed` ➔ `Danh từ + V3/ed`

### 💡 10 Ví dụ thực tế:

1.  *Đầy đủ:* "This is the library **which was selected** by our team."
    ➔ **Rút gọn: "This is the library selected by our team."** *(Đây là thư viện được chọn bởi team chúng tôi).*
2.  *Đầy đủ:* "The changes **that are deployed** yesterday are working fine."
    ➔ **Rút gọn: "The changes deployed yesterday are working fine."** *(Các thay đổi được deploy hôm qua đang chạy tốt).*
3.  *Đầy đủ:* "Please review the code **that was written** by Huy."
    ➔ **Rút gọn: "Please review the code written by Huy."** *(Vui lòng review code được viết bởi Huy).*
4.  *Đầy đủ:* "The features **that were requested** by the client are completed."
    ➔ **Rút gọn: "The features requested by the client are completed."** *(Các tính năng khách hàng yêu cầu đã được hoàn thành).*
5.  *Đầy đủ:* "All logs **which are stored** in the folder are encrypted."
    ➔ **Rút gọn: "All logs stored in the folder are encrypted."** *(Tất cả log lưu trong thư mục đều được mã hóa).*
6.  *Đầy đủ:* "The API keys **that are used** for authentication have expired."
    ➔ **Rút gọn: "The API keys used for authentication have expired."** *(Các API key dùng để xác thực đã hết hạn).*
7.  *Đầy đủ:* "The database tables **which were created** last week need schema updates."
    ➔ **Rút gọn: "The database tables created last week need schema updates."** *(Các bảng database tạo tuần trước cần cập nhật schema).*
8.  *Đầy đủ:* "This is the configuration **that is recommended** by the DevOps team."
    ➔ **Rút gọn: "This is the configuration recommended by the DevOps team."** *(Đây là cấu hình được khuyến nghị bởi team DevOps).*
9.  *Đầy đủ:* "Only files **that are committed** to the main branch are deployed."
    ➔ **Rút gọn: "Only files committed to the main branch are deployed."** *(Chỉ những file được commit lên branch main mới được deploy).*
10. *Đầy đủ:* "The server certificates **which were renewed** today are working."
    ➔ **Rút gọn: "The server certificates renewed today are working."** *(Các chứng chỉ server được gia hạn hôm nay đang hoạt động tốt).*

## 4. Mẹo 3: Liên kết câu bằng ", which" (Mệnh đề quan hệ nối tiếp)

Hãy dùng cụm `, which` (luôn có dấu phẩy đứng trước) ở cuối câu để bình luận hoặc chỉ ra hệ quả cho **toàn bộ sự việc bạn vừa nói ở vế trước**. Đây là cách nối ý cực kỳ chuyên nghiệp trong các buổi họp.

### 💡 10 Ví dụ thực tế:

1.  **"The server went offline at midnight, which caused the database connection to fail."** *(Server bị sập lúc nửa đêm, điều này làm cho kết nối database bị lỗi).*
2.  **"The client decided to change the designs, which means we have to rewrite the CSS."** *(Khách hàng quyết định đổi thiết kế, điều đó đồng nghĩa với việc chúng ta phải viết lại CSS).*
3.  **"We forgot to run the migrations on staging, which led to a schema mismatch error."** *(Chúng tôi quên chạy migration trên staging, dẫn đến lỗi lệch schema).*
4.  **"The API token expired early, which blocked the automated test suite."** *(Token API hết hạn sớm, làm chặn đứng bộ test tự động).*
5.  **"Huy refactored the query logic, which improved the response speed by 40%."** *(Huy đã refactor logic truy vấn, điều này cải thiện tốc độ phản hồi 40%).*
6.  **"The user session doesn't refresh automatically, which causes security warnings."** *(Session của user không tự động reload, điều này gây ra các cảnh báo bảo mật).*
7.  **"We resolved the memory leak, which stabilized the server load."** *(Chúng tôi đã sửa xong lỗi rò rỉ bộ nhớ, điều này làm ổn định tải của server).*
8.  **"The payment portal went down, which resulted in aborted user checkouts."** *(Cổng thanh toán bị sập, dẫn đến việc người dùng bị hủy checkout).*
9.  **"The package was upgraded yesterday, which introduced a breaking dependency."** *(Package được nâng cấp hôm qua, điều này gây ra lỗi tương thích).*
10. **"We moved our assets to a CDN, which dramatically decreased initial load times."** *(Chúng tôi chuyển asset sang CDN, nhờ đó giảm đáng kể thời gian tải trang đầu tiên).*

## 5. Bảng so sánh nhanh: Đầy đủ vs Rút gọn

| Mệnh đề đầy đủ | Bản rút gọn tự nhiên | Nghĩa tiếng Việt |
| :--- | :--- | :--- |
| The PR **that contains** the bug... | The PR **containing** the bug... | PR có chứa lỗi... |
| The issue **which was reported** by user... | The issue **reported** by the user... | Lỗi được báo cáo bởi user... |
| The API **that runs** in the background... | The API **running** in the background... | API chạy ngầm... |
| The server **which is hosted** on AWS... | The server **hosted** on AWS... | Server được host trên AWS... |
| The framework **which was chosen**... | The framework **chosen**... | Framework được chọn... |
