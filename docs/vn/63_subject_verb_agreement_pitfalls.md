# Lỗi Hòa Hợp Chủ Vị Trong Ngành IT (Daily English)

Lỗi chia động từ không hòa hợp với chủ ngữ (Subject-Verb Agreement) có thể làm cho email hoặc tài liệu của bạn thiếu đi sự chuyên nghiệp. Trong ngành IT, chúng ta thường xuyên dùng các danh từ không đếm được như *data*, *code*, và *software*, vốn có các quy tắc chia động từ số ít rất nghiêm ngặt.

---

## 1. Các Danh Từ Không Đếm Được Trong IT (Luôn chia Động từ số ít)

Các danh từ dưới đây là **danh từ không đếm được (uncountable)** trong tiếng Anh. Chúng **không bao giờ** được thêm "s" ở cuối, và **luôn luôn** đi với **động từ số ít**.

### 💡 Mẹo 1: Bộ ba từ số ít kinh điển (Software, Code, Data)
- **software:** (Tuyệt đối không nói *softwares*).
- **code:** (Không nói *codes* khi ám chỉ các dòng code. Nếu muốn dùng số nhiều, hãy nói *lines of code*).
- **data:** (Trong ngành IT hiện đại, *data* được coi là một danh từ khối không đếm được, đi với động từ số ít).

### 💡 10 Ví dụ thực tế:

1.  **"This software needs to be updated immediately."** *(Phần mềm này cần được cập nhật ngay lập tức - Không dùng: need).*
2.  **"The code looks clean and well-structured."** *(Đoạn code trông rất sạch sẽ và cấu trúc tốt - Không dùng: look).*
3.  **"The migration data is loaded into the staging database."** *(Dữ liệu migration đã được load vào database staging - Không dùng: are).*
4.  **"New equipment is arriving for the server room tomorrow."** *(Thiết bị mới sẽ được giao cho phòng server vào ngày mai - Không dùng: are / equipments).*
5.  **"Confidential information is protected by our encryption algorithms."** *(Thông tin bảo mật được bảo vệ bởi thuật toán mã hóa của chúng ta - Không dùng: informations / are).*
6.  **"All software running in our containers requires regular security updates."** *(Tất cả phần mềm chạy trong container của chúng ta đều yêu cầu cập nhật bảo mật thường xuyên).*
7.  **"The source code contains several unoptimized loops."** *(Mã nguồn chứa một vài vòng lặp chưa được tối ưu).*
8.  **"Historical log data is compressed to save disk space."** *(Dữ liệu log lịch sử được nén lại để tiết kiệm dung lượng đĩa).*
9.  **"Network hardware is monitored 24/7 by the DevOps team."** *(Phần cứng mạng được theo dõi 24/7 bởi team DevOps).*
10. **"Your feedback information helps us improve the user interface."** *(Thông tin phản hồi của bạn giúp chúng tôi cải thiện giao diện người dùng).*

## 2. Bẫy Chia Động Từ Với "One of / Each of / None of"

Khi câu bắt đầu bằng các cụm từ này, lập trình viên hay chia động từ số nhiều do danh từ đứng sau nó là số nhiều. Đây là lỗi sai ngữ pháp nghiêm trọng.

### 💡 Mẹo 2: Tập trung vào Đại từ chính, không nhìn danh từ số nhiều đứng gần
- **One of + Danh từ số nhiều + Động từ số ít:**
  - ❌ *Sai:* "One of the database instances are offline."
  - 🟢 *Đúng:* **"One of the database instances is offline."** *(Một trong các instance database đang ngoại tuyến - Chủ ngữ là "One", không phải "instances").*
- **Each of + Danh từ số nhiều + Động từ số ít:**
  - ❌ *Sai:* "Each of these API endpoints have a rate limit."
  - 🟢 *Đúng:* **"Each of these API endpoints has a rate limit."** *(Mỗi endpoint API này đều có giới hạn rate limit - Chủ ngữ là "Each").*
- **None of + Danh từ số nhiều + Động từ số ít/số nhiều:** (Trong văn viết trang trọng, động từ số ít được ưu tiên).
  - 🟢 **"None of the microservices is affected by the vulnerability."** *(Không có dịch vụ microservice nào bị ảnh hưởng bởi lỗ hổng bảo mật).*

### 💡 10 Ví dụ thực tế:

1.  **"One of the server nodes is experiencing high latency."** *(Một trong các node server đang bị độ trễ cao).*
2.  **"Each of our team members has access to the AWS console."** *(Mỗi thành viên trong team đều có quyền truy cập vào console AWS).*
3.  **"None of the security vulnerabilities has been exploited."** *(Không có lỗ hổng bảo mật nào bị khai thác).*
4.  **"One of the configuration files is missing from the directory."** *(Một trong các file cấu hình bị thiếu khỏi thư mục).*
5.  **"Each of the pull requests requires two approvals before merging."** *(Mỗi pull request đều yêu cầu hai người duyệt trước khi merge).*
6.  **"None of the legacy databases contains user credentials."** *(Không có database cũ nào chứa thông tin đăng nhập của user).*
7.  **"One of the QA testers is verifying the checkout flow now."** *(Một trong các QA tester đang xác thực luồng checkout).*
8.  **"Each of these functions executes in a separate thread."** *(Mỗi hàm này đều thực thi trong một luồng độc lập).*
9.  **"None of the refactored code needs further optimization."** *(Không có phần code nào được refactor cần tối ưu thêm nữa).*
10. **"One of our core modules is written in Rust."** *(Một trong các module cốt lõi của chúng ta được viết bằng Rust).*

## 3. Chia Động Từ Với Từ Nối (As well as, Along with, Neither/Nor)

### 💡 Mẹo 3: Quy tắc "Đóng ngoặc đơn" (As well as, Along with)
Các cụm từ như *as well as*, *along with*, và *in addition to* không tạo ra chủ ngữ số nhiều. Hãy coi chúng như một phần bổ sung nằm trong dấu ngoặc đơn; động từ chỉ chia theo **chủ ngữ chính đứng đầu tiên**.
- ❌ *Sai:* "The lead architect, along with his developers, are reviewing the PR."
- 🟢 *Đúng:* **"The lead architect, along with his developers, is reviewing the PR."** *(Lead architect, cùng các dev của anh ấy, đang review PR - Động từ chia theo "architect").*

### 💡 Mẹo 4: Quy tắc "Gần nhất" (Either... or / Neither... nor)
Khi dùng cấu trúc lựa chọn *either/or* hoặc phủ định kép *neither/nor*, động từ sẽ chia theo **danh từ đứng gần nó nhất**.
- 🟢 **"Neither the backend server nor the database instances are running."** *(Cả server backend lẫn các DB instance đều không chạy - Động từ chia theo "instances" số nhiều).*
- 🟢 **"Neither the database instances nor the backend server is running."** *(Động từ chia theo "server" số ít).*

### 💡 10 Ví dụ thực tế:

1.  **"The tech lead, as well as the product owners, agrees with the timeline."** *(Tech lead, cũng như các product owner, đồng ý với timeline).*
2.  **"The migration script, along with the backup files, is stored in S3."** *(Script migration, cùng với các file backup, được lưu trữ trong S3).*
3.  **"Either the database host or the API servers are down."** *(Hoặc là host database hoặc là các server API bị sập).*
4.  **"Neither the primary server nor the secondary replicas are responding."** *(Cả server chính lẫn bản replica phụ đều không phản hồi).*
5.  **"Huy, in addition to the frontend team, is working on dark mode."** *(Huy, bên cạnh team frontend, đang làm tính năng dark mode).*
6.  **"The documentation, as well as the API examples, needs updating."** *(Tài liệu hướng dẫn, cũng như các ví dụ API, cần được cập nhật).*
7.  **"Either the database credentials or the port number is incorrect."** *(Hoặc là thông tin đăng nhập database hoặc là số cổng bị sai).*
8.  **"Neither the environment variables nor the config file is loaded."** *(Cả các biến môi trường lẫn file cấu hình đều chưa được load).*
9.  **"The manager, along with the client stakeholders, is reviewing the roadmap."** *(Quản lý, cùng với các bên đối tác phía khách hàng, đang xem lại roadmap).*
10. **"Neither the main domain nor the subdomains are working."** *(Cả tên miền chính lẫn các tên miền phụ đều đang không hoạt động).*

## 4. Bảng Tra Cứu Nhanh

| Danh từ / Cụm từ | Động từ chia đúng | Ví dụ |
| :--- | :--- | :--- |
| **The data** | Số ít | "The data **is** encrypted." |
| **All software** | Số ít | "All software **requires** licensing." |
| **Our code** | Số ít | "Our code **runs** in Docker." |
| **One of the APIs** | Số ít | "One of the APIs **is** failing." |
| **Each developer** | Số ít | "Each developer **has** a repository." |
| **None of the bugs** | Số ít | "None of the bugs **is** critical." |
| **Huy, along with team** | Số ít | "Huy, along with team, **is** preparing." |
| **Either DB or servers** | Số nhiều | "Either DB or servers **are** down." |
